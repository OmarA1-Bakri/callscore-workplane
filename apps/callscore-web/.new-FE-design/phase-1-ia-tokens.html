Appendix A · binding acceptance flows

Seven unassisted-user tasks mapped to surface paths.

Each task below is a binding acceptance flow App. A. The path shows the exact surface transitions an unassisted user takes. Gated steps are marked [gate]; no task requires a gated step to complete on free tier.

Task 01 · top 90d performer, credible N
Find the top-performing creator in the last 90 days with a credible sample size.
Leaderboard /→Timeframe = 90d→Filter: Sample-size ≥ 30→Sort by Alpha ↓→Row #1 · creator profile
Unassisted because: filter chip says "Min sample N" with explainer hover; default timeframe switcher is visible above table. §8 · §14
Task 02 · explain ranking
Explain why a creator ranks highly.
Profile header→"Why they rank here" panel→Score explanation drawer→Factor weights · inputs · evidence
Unassisted because: plain-language sentence above the fold; drawer opens from every Alpha Score. §5B · §7
Task 03 · verify one call
Verify the evidence behind one scored call.
Profile recent-calls→Row click→Call detail drawer→Source link + archive→Resolution logic visible
Provenance-indicator is a first-class UI primitive, present on every call row. §7 · §8 calls-layer
Task 04 · active bullish consensus
Find an active bullish consensus signal.
Nav · Signals→Active / Bullish column→Sort by confidence ↓→Signal card→Constituent calls
Consensus snapshot on leaderboard also exposes top 3 live without navigation. §5A · §5D
Task 05 · compare two creators
Compare two creators.
Leaderboard→Row checkbox × 2→Compare bar appears→Open comparison→Side-by-side metrics + chart overlay
Free tier allows 2-way compare. 3rd selection triggers §16.3 upgrade. §8 · §15 · §16.3
Task 06 · set an alert
Set or attempt to set an alert.
Profile header · Alert CTA→Free: Follow = basic account notif→Configurable types [gate]→§16.2 upgrade prompt · in-context
"Attempt to set" succeeds on free at the follow level; configurable types are the paid moment — value demonstrated before gate. §15 · §16.2 · §5F
Task 07 · understand premium
Understand what premium unlocks.
Any in-context gate→Teaser shows locked value→"See everything premium" link→Pricing / feature matrix
Preview-with-lock, never a wall; every gate anchors to the user's present intent. §13 · §15 · §16

II

Design tokens

Covers §11 · §13 · §14 · §9

Direction committed: monochrome warm-neutral base with a single ochre accent. Editorial serif for titles and metric heroes; sharp sans for body; mono for every datum. Flat card system, border-led elevation, motion only where it signals state change §11.

Tension resolved up-front: editorial sharpness vs perceived bias §9 — confidence comes from typography and restraint, not color. One accent ≠ hype; severity comes from copy, not red flashing. Density vs readability §9 — density is a user-chosen mode (compact / comfortable / relaxed), not a default imposed. Default is comfortable.

Six UX tensions, resolved up-front.

Every surface inherits these resolutions

The brief requires each tension to be named and resolved rather than drifted past. Phase 1 records the resolution at the token level so every component in Phase 2 inherits the decision §9.

Density vs Readability
Density is a user-chosen mode, not an imposed default. Three row heights; comfortable default.
Tokens: --row-compact/comfortable/relaxed. Saved in localStorage; restored on return. §8 · §9
Confidence vs Overclaiming
Confidence is typographic — serif italic, tabular-num, restraint. Not chromatic.
Score at low-N rendered at 60% opacity + Low N badge. Never hidden. §13 · §14
Gating vs Frustration
Preview-with-lock. Every gate shows value and anchors to intent. No walls, no interstitials.
--lock token; blur(6px) + value-prop chip pattern; six §16 moments mapped. §15 · §16
Explainability vs Clutter
Metrics self-decode on the surface; methodology page is reference, not a prerequisite.
Every §14 metric has an inline explanation pattern (hover reveal). Specified in Phase 2. §14
Freshness vs Noise
Subtle delta indicators (4px left-border, --new). No badges that pulse or animate.
24h TTL on "new"; stale data kept visible and labeled, not hidden. §13
Editorial sharpness vs Perceived bias
Sharpness comes from copy and typography; evidence path is one interaction from every judgment.
§7 provenance is a first-class primitive. "Severe with bad performers, fair with evidence." §3 · §7
Δ

Direction confirmation before Phase 2.

Per engagement working method

Confirm the direction, or name what to re-draft.

Phase 1 presents the IA and the token system together because decisions in one constrain the other — for example, the provenance pattern (§7) must be renderable inside every nine-state row, and the three density modes must be legible in the same mono at the same tabular-num settings. The three open directional choices are:

  1. Accent hue — ochre (#C9A24B)Editorial gold that reads as judgment, not retail-trading gold. Alternatives staged: slate-blue (reads as analytical, loses severity), desaturated amber (reads as warning, bleeds state-color). Recommend proceeding with ochre.
  2. Editorial serif — Source Serif 4Pairs with Inter Tight without magazine-luxury drift. Alternative: GT Sectra (stronger contrast, riskier licensing). Recommend Source Serif 4 for free-tier and licensing clarity.
  3. Default density — comfortable (36px)Balances §9 tension without forcing terminal density on first-time visitors. Alternatives: compact (alienates new users), relaxed (undersells the §0 density thesis). Recommend comfortable with compact available under Tweaks / preferences.
