D5 · A single popover for every metric — same anatomy, ten metrics, every surface.

surfaces · 5

Across Leaderboard / Profile / Signals / Compare / Dashboard, users see the same numbers over and over — α, β-α, CI, tier-weighted N, win rate, retraction rate, first-mover rate, median horizon, velocity, overlap-with-you. Right now each surface explains them differently (or not at all). D5 fixes that with one popover component, applied ten times. Hover any metric → same anatomy, same tone, same evidence trail.

Design bet: metrics people can't explain, people don't trust. A 4-part popover — what it is · how we compute it · what's good/bad · where the numbers come from — is the cheapest possible trust investment. A 5th optional slot (caveat) surfaces when the metric is fragile: low N, wide CI, recent-only, etc.

What · plain-English definition, one sentence. Never a formula here. Always reads as "X is Y, so positive/high = Z".

4

02 · β-α · Beta-adjusted alphagate.beta.column · Pro
Metric · risk-adjusted performance
β-α Beta-α avg · 90d
What
α after stripping out what was explainable by general crypto-market movement. Isolates skill from regime luck.
How
β-α = α − (β · r_market). β from 30-day rolling regression vs. CMC Top-10 index.
Good
−3+1.5 typical+3 elite
Sourceα ledger × rolling β · 30d window · β spec
Unstable during regime changes (β re-estimates). Treat as directional for 90d.
03 · CI · Confidence intervalconfidence input
Metric · how sure are we
CI 95% interval bootstrap · 1000x
What
The range α would land in if we re-sampled this creator's calls 1,000 times. Narrow = stable. Wide = few or noisy calls.
How
Bootstrap resample of resolved calls, 1000 iterations, 95% percentile band reported as [α_low, α_high].
Good
±0.8 tight±2.0 wide±4.0 flag
SourceResolved calls · per-creator bootstrap · explain
Always visualized as the shaded band behind α on profile charts. Never collapse to ±.
04 · N · Tier-weighted Nsample size
Metric · sample adequacy
N Tier-weighted calls 90d window
What
Resolved calls in the window, weighted by tier — Tier-S calls count 1.0, Tier-A 0.7, Tier-B 0.4. Penalizes creators who only make throwaway predictions.
How
N_eff = Σ w(tier_i) over resolved calls in window. w = {S:1.0, A:0.7, B:0.4, C:0.2}.
Good
<12 low30 adequate80+ rich
SourceCall ledger × tier registry · tier criteria
Below 12: α is shown in gray italic + "low-N" tag — do not rank on it.
05 · WR · Win ratesupporting
Metric · hit rate on direction
WR Win rate %, 90d
What
Share of resolved calls where the direction was right — regardless of size. Simple, but easy to game with small moves.
How
WR = #{rᵢ > 0} / N over resolved calls. Not weighted.
Good
45% low58% typical70% elite
SourceCall ledger · directional outcome · resolution logic
High WR + low α = small wins, big losses. Always pair with α.

Share of calls the creator publicly revised or canceled before resolution. Counterintuitively, higher is healthier — means they update on new evidence.

FM First-mover rate %, 90d
What
Share of this creator's calls where they were first to voice the thesis that later became consensus. Distinguishes leaders from echoes.
How
FM = #{first by >2h on 2+ aligned thesis} / thesis count. 2h lead threshold to ignore simultaneous posts.

08 · H · Median horizonstyle context
Metric · trading style
H Median horizon days to resolution
What
Typical time between call and resolution. Characterizes style — scalpers, swing, position.
How
Median of t_resolved − t_call over resolved calls in 90d. Median, not mean, to resist outliers.
Good
Not better or worse — just context. Match horizon to your own holding period.
<3d scalper · 3-10d swing · >10d position
SourceCall ledger timestamps · resolution rules
09 · V · Velocity NEWrecency proxy
Metric · publishing cadence
V Velocity calls / week
What
How many directional calls this creator publishes per week, over the last 30 days. Helps distinguish active from dormant, independent of historical α.
How
V = N_calls_30d / 4.3. Only includes calls that will eventually resolve (directional + asset-specific).
Good
Context, not rank. Too high (>15/wk) = noise risk. Too low (<0.5/wk) = dormant.
1-4/wk healthy cadence
SourceCall ledger · last 30d · what counts
Dormancy flag fires at V < 0.3 for 14d — surfaced on profile as stale.
10 · OL · Overlap-with-you NEWpersonal
Metric · thesis agreement with your positions
OL Overlap-with-you %, 30d
What
Share of this creator's active theses that align with assets you currently hold (from connected wallet or manual portfolio).
How
OL = #{thesis on your asset ∩ your direction} / thesis count. Only active, not resolved.
Good
High ≠ better. High overlap = echo chamber risk; low overlap = diversifying voice.
Use as a filter, not a ranker.

04 · behavior

Trigger rules · the same everywhere.

desktop · hover + focus
mobile · tap · tap-away
a11y · keyboard + screenreader
Trigger
Desktop: hover (180ms delay) or Tab + focus. Mobile: tap (pop anchors above tap point, tap-away to dismiss). Never on-click-navigation — the trigger element stays navigable.
Dismiss
Desktop: mouse-out + 120ms grace. Mobile: tap outside, scroll, or Esc. Auto-dismiss on any route change.
Position
Anchored to trigger, 8px offset, arrow toward trigger. Flips to opposite side near viewport edges. Never covers the number it's explaining.
Value binding
Header .av slot renders the current context value — if user is looking at @CryptoInsights' α on 90d, the popover header reads avg · 90d. Same metric, different context → different popover header, same anatomy.
Scale calibration
The "good" scale marker shows the user's current value on the typical-to-elite range. So on @CryptoInsights (+8.42) the α popover's marker sits near the right edge; on a Tier-B creator (+0.3) it sits near the middle. Calibration per-view, anatomy fixed.
Caveat firing
Caveat slot appears when this specific instance is fragile: N < 12, CI width > 4.0, last call > 14d. On a healthy @CryptoInsights α value, no caveat; on a new creator's α, caveat appears automatically.
Persistence
Popover is ephemeral, not pinned. But a ℹ︎ button inside the popover opens full methodology drawer — same one linked from the footer source row. One source of truth for the full math.
Accessibility
Trigger has aria-describedby pointing at popover. Popover is role="tooltip". Focus returns to trigger on Esc. Screenreader reads "α, 8.42 — Alpha, leaderboard rank driver. Log-return excess..." — same 4 slots in order.
