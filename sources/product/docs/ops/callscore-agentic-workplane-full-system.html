What makes these proper agents, not cron jobs

  1. Identity: named specialist role such as marketing-channel-growth, content_creator, marketing-compliance-linter, whop_auto.
  2. Bounded authority: each agent owns a precise lane and cannot cross into sends, spend, DB, deploy, Whop/provider mutation, or secrets without gates.
  3. Governance: Workplane + GTM registry + Whop/Composio gates decide what can execute.
  4. Memory: handovers, receipts, state_manager/FalkorDB/agentmemory/callscore-memory preserve durable context.
  5. Cadence + constraints: agents run in controlled loops/cadences; cooldowns and approval requirements stop unsafe repetition.
  6. Taste: persona/compliance/humanizer/quality review define what good output looks like and what must be rejected.

How I control it

  1. Registry first: load docs/ops/callscore-gtm-agent-registry.json; it defines owner, provider, allowed/forbidden actions, gates, receipts, rollback, status, and next safe action.
  2. Workplane second: run npm run workplane:status; it returns readiness domains, job specs, safe command boundaries, and next autonomous action.
  3. Evidence before mutation: run live public verify, freshness, audit, tests, dry-runs, and read-only app inventory.
  4. Execute only safe lanes: owned public organic posts can run when READY_PUBLIC_OWNED and safe; dangerous actions need exact approval + receipt + rollback.
  5. Record every state change: receipts under .tmp/workflow-receipts/, handover updates, docs, and git commits for durable control artifacts.

What remains impossible without approval

I do not mutate paid/provider/customer/data/deploy surfaces from ambition or convenience. The control plane blocks them until the matching gate, payload/diff, rollback path, and receipt exist.

  • paid spend, ads, boosts, paid APIs/LLMs/SaaS
  • email, DM, newsletter, outreach, non-owned community posts
  • Whop pricing/product/customer/payment/entitlement/provider changes
  • CRM/analytics/provider writes
  • DB writes/backfills/recomputes, deploys, infra/webhooks
  • credential rotation, secret exposure, destructive actions

Website / data plane

Netlify serves call-score.com. Public/API reads flow to the HH Read API and local HH PostgreSQL. Vercel and Neon are stale/non-production. Public data is filtered to public-safe ranked rows.

Pipeline / ML plane

Creator/video/transcript data becomes calls through extraction, symbol/outcome matching, scoring, freshness checks, and audits. Local Ollama Gemma/Qwen runs shadow and diff workflows; write-canary or broad promotion stays gated.

Commercial plane

Whop Auto proves checkout/entitlement readiness and can run provider health, plan inventory, entitlement dry-runs, webhook safe replay, customer checks, and activation reviews. Financial/provider mutations stay fail-closed.

GTM plane

Art of War generates strategy, content, persona/compliance checks, campaign dossiers, dry-runs, receipts, and owned public canaries. Safe owned organic posts may execute; sends/spend/non-owned/restricted claims do not.

Connected app plane

Composio is the first-class third-party access layer for X/Twitter, LinkedIn, Gmail, Discord, Attio, PostHog, and Hugging Face. Tool availability does not override gates.

Governance plane

Hermes plus Workplane coordinates status, job specs, cooldowns, receipts, approval review, handovers, memory, validation, and exact rollback instructions.
